2. REST API의 기본 개념

'REST'는 가장 인기 있는 '웨이터 고용 방식' (아키텍처 스타일)입니다.
"자원의 상태(정보)를 전달한다"는 의미를 가집니다.

1. 자원 (Resource)

API로 다루는 모든 것 (명사)

/users /posts/1

2. 행위 (Verb)

자원에 대해 수행할 동작 (동사)

GET POST PUT DELETE

3. 표현 (Representation)

자원의 상태를 나타내는 형식

{ "name": "홍길동" }

3. REST API 사용법과 규약 (CRUD)

REST API는 4가지 기본 동작(CRUD)을 기반으로 합니다.
아래 탭을 클릭하여 각 행위(Method)가 어떻게 사용되는지 확인해 보세요.

POST: 새로운 자원을 생성합니다.

새로운 유저를 생성하거나 새 게시글을 작성할 때 사용합니다. 요청 본문(Body)에 생성할 데이터(JSON)를 담아 보냅니다.

POST /users

GET: 자원을 조회합니다.

모든 유저 목록을 보거나, 특정 ID를 가진 유저 1명의 정보를 볼 때 사용합니다. 데이터를 가져오기만 할 뿐, 서버에 변경을 일으키지 않습니다.

GET /users (목록 조회)
GET /users/123 (상세 조회)

PUT / PATCH: 기존 자원을 수정합니다.

특정 유저의 이름이나 이메일 주소를 변경할 때 사용합니다. PUT은 전체 데이터를 교체, PATCH는 일부 데이터만 수정하는 방식입니다.

PUT /users/123

DELETE: 자원을 삭제합니다.

특정 ID의 유저를 삭제하거나 게시글을 지울 때 사용합니다.

DELETE /users/123

4. REST API는 어디에 쓰일까?

API는 서버(데이터)와 클라이언트(화면)를 연결하는 '전기 플러그'와 같습니다.
서버는 API라는 '콘센트'를 제공하고, 다양한 기기들이 '플러그'를 꽂아 데이터를 사용합니다.

📱 모바일 앱 (iOS/Android)
💻 웹 프론트엔드 (Vue/React)
🤖 외부 서비스 (카카오/네이버)
🔌 REST API 서버 (Spring Boot) 🔌

5. [실전 1] Spring Boot로 API 서버 만들기

Spring Boot의 @RestController를 사용하면 누구나 쉽게 API 서버를 만들 수 있습니다. 

@RestController
@RequestMapping("/api/users")
public class UserController {

    // [GET] /api/users/1
    @GetMapping("/{id}")
    public UserDto getUserById(@PathVariable Long id) {
        // ID로 유저를 찾는 로직...
        UserDto user = new UserDto(id, "홍길동", "user@test.com");
        return user; // 객체를 리턴하면 JSON으로 자동 변환됩니다.
    }

    // [POST] /api/users
    @PostMapping
    public UserDto createUser(@RequestBody CreateUserRequest request) {
        // 유저를 저장하는 로직...
        UserDto newUser = new UserDto(1L, request.getName(), request.getEmail());
        return newUser;
    }
}

6. [실전 2] Vue 3에서 API 호출하기

프론트엔드에서는 axios 라이브러리를 사용해 Spring 서버에 데이터를 요청합니다. 

<script setup>
import { ref, onMounted } from 'vue';
import axios from 'axios';

const user = ref(null);

// 컴포넌트가 로드될 때 API 호출 (GET 요청)
onMounted(async () => {
  try {
    const response = await axios.get('http://localhost:8080/api/users/1');
    user.value = response.data;
  } catch (error) {
    console.error("API 호출 실패:", error);
  }
});

// 버튼 클릭 시 API 호출 (POST 요청)
const createUser = async () => {
  const newUser = { name: '새 유저', email: 'new@test.com' };
  const response = await axios.post('http://localhost:8080/api/users', newUser);
  alert('유저 생성 성공: ' + response.data.name);
};
</script>

<template>
  <div v-if="user">
    <h1>{{ user.name }}</h1>
    <p>{{ user.email }}</p>
  </div>
  <button @click="createUser">유저 생성</button>
</template>

7. [협업] API 명세서 (Swagger)

API 명세서는 프론트엔드와 백엔드 개발자 간의 '공식 메뉴판'입니다.
요청 주소, 필요한 값, 응답 데이터 형식을 정확히 기록해야 합니다.
요즘은 Swagger (OpenAPI)를 사용해 코드를 기반으로 명세서를 자동 생성합니다.
